#  서울 지하철 혼잡도 분석 (Seoul Subway Congestion Analysis)

이 프로젝트는 서울시 공공 데이터를 활용하여 지하철 호선별, 시간대별 혼잡도를 분석합니다.

본 프로젝트의 목표는 데이터를 전처리하고 시각화하여, 서울 지하철의 주요 혼잡 패턴(특히 출퇴근 시간)을 도출하고 이를 통해 유의미한 인사이트를 발견하는 것입니다.

---

## 1. 사용 데이터 (Data Source)

* **Data:** 서울교통공사_지하철혼잡도정보
* **Source:** 공공데이터포털 (data.go.kr)

---

## 2. 사용 기술 (Tools & Technologies)

* **Language:** Python
* **Libraries:** Pandas, Numpy, Matplotlib, Seaborn
* **Environment:** Jupyter Notebook

---

## 3. 분석 과정 및 핵심 결과 (Analysis & Findings)

### 3-1. 주중 호선별 평균 혼잡도 (Average Weekday Congestion by Line)

먼저, 주중(Weekday) 데이터를 기준으로 각 호선의 평균 혼잡도를 비교했습니다.

<img width="1000" height="553" alt="image" src="https://github.com/user-attachments/assets/1e4767f2-b8ff-46b9-be12-7f561c549e90" />


* **Insight:** 분석 결과, **Line 2 (2호선)**가 다른 호선에 비해 압도적으로 평균 혼잡도가 높음을 알 수 있습니다. 이는 2호선이 서울의 주요 업무 지구(강남, 구로디지털단지)와 대학가를 모두 순환하는 핵심 노선이기 때문으로 보입니다.

### 3-2. 2호선 시간대별 혼잡도 (Weekday Congestion Pattern: Line 2)

가장 혼잡한 2호선을 대상으로, 주중 시간대별 혼잡도 추이를 라인 플롯으로 시각화했습니다.

## 4. 결론 및 제언 (Conclusion & Future Work)

### 4-1. 핵심 요약 (Key Takeaways)

* 주중에는 **전형적인 출퇴근 'M'자 패턴**이 나타났으며, 주말에는 이 피크가 사라지고 오후 시간대에 완만한 혼잡도를 보였습니다.
* 모든 호선 중 **Line 2(2호선)**가 주중/주말과 상관없이 가장 혼잡한 노선임을 데이터를 통해 입증했습니다.
* 히트맵 분석 결과, 특정 시간대(e.g., `[18:30]`)에는 `[Line 2, Line 7]` 등 다수 노선의 혼잡도가 매우 높음을 확인했습니다.

### 4-2. 분석의 한계점 (Limitations)

1.  **공휴일 데이터의 부재:** 데이터가 '평일', '토요일', '일요일'로만 구분되어, '평일이지만 공휴일'인 경우의 데이터가 '주중' 평균에 포함되어 분석 결과를 왜곡했을 가능성이 있습니다.
2.  **방향성(상/하선) 미고려:** 이번 분석에서는 '호선' 단위의 평균을 보기 위해 상/하선 데이터를 통합했습니다. 이로 인해 '내선순환'과 '외선순환'의 시간대별 혼잡도 차이를 확인하지 못했습니다.

### 4-3. 향후 개선 과제 (Future Work)

* **지도 시각화(Geospatial Analysis):** 역별 위도/경도 데이터를 추가하여, `Folium` 라이브러리를 활용해 **서울 지도 위에 시간대별 혼잡도를 시각화**하여 직관성을 높일 수 있습니다.
* **심층 분석:** '상/하선'을 구분하여 어느 방향(e.g., 강남으로 향하는 방향)이 더 붐비는지 분석하거나, '환승역' 여부에 따른 혼잡도를 비교 분석할 수 있습니다.
